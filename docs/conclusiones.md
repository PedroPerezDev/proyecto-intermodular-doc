# Conclusiones

![Imagen decorativa](img/logos4.jpg)

---

## Dificultades encontradas

Durante la realización de este proyecto se han detectado algunas dificultades:

- Aprender la sintaxis de **Markdown** correctamente  
- Mantener la **estructura del repositorio** organizada  
- Integrar imágenes y enlaces relativos sin errores  
- Entender cómo funciona **GitHub Pages** para publicar la documentación

---

## Aprendizajes

Gracias a esta práctica se han adquirido conocimientos importantes:

- Redacción de documentación técnica clara y estructurada  
- Uso de **listas, tablas, imágenes y enlaces** en Markdown  
- Gestión de un repositorio en **GitHub**  
- Publicación de documentación web mediante **GitHub Pages**  
- Mejores prácticas en la organización de un proyecto software

---

## Posibles mejoras

Aunque TaskFlow es un proyecto educativo, se podrían implementar mejoras como:

- Añadir funcionalidades interactivas reales (CRUD de tareas)  
- Incorporar un backend para guardar información de manera persistente  
- Mejorar el diseño visual con CSS avanzado o frameworks como Tailwind  
- Documentar los casos de prueba y resultados de testing
