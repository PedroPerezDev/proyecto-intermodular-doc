# Arquitectura del proyecto

![Diagrama de arquitectura](img/logos3.jpg)

---

## Estructura general

TaskFlow sigue una arquitectura sencilla orientada a **proyectos web educativos**, organizada de la siguiente manera:

- **Frontend**
  - HTML5 para la estructura
  - CSS3 para el diseño y estilo
  - JavaScript para la lógica de interacción
- **Repositorio**
  - Control de versiones con Git
  - Documentación en Markdown en `docs/`
  - Imágenes en `docs/img/`

---

## Flujo de información

1. El usuario visualiza la interfaz en su navegador
2. Interactúa con tareas (crear, completar, organizar)
3. La aplicación actualiza el estado de las tareas (simulado para la práctica)
4. Todo se refleja en la interfaz y en la documentación correspondiente
